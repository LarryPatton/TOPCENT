# 销售产品分类表与图片资料规范

本文档用于规范销售、产品或非技术人员提供给网站的 Excel 和图片资料。

## Excel 表格规范

请维护 Excel 第一张表：

```text
产品分类表0314调整版
```

网站主要读取以下字段：

```text
C：一级分类英文
D：一级分类中文
E：二级分类英文
F：二级分类中文
G：三级分类英文
H：三级分类中文
I：型号
K：备注
```

## 分类填写规则

一级分类必须保持固定顺序。当前网站使用以下 15 个一级分类：

```text
Slide Solutions / 滑轨解决方案
Hinge Solutions / 铰链解决方案
Kitchen Solutions / 厨柜解决方案
Wardrobe Solutions / 衣柜解决方案
Function Solutions / 功能解决方案
Sliding Solutions / 移门解决方案
Flap Solutions / 翻门解决方案
Handle Solutions / 拉手解决方案
Lighting Solutions / 照明解决方案
Open-close Solutions / 开合解决方案
Connector Solutions / 连接解决方案
Decoration Solutions / 装饰解决方案
Feet Solutions / 基座解决方案
Frame Solutions / 框架解决方案
Door Solutions / 门配解决方案
```

二级分类和三级分类尽量保持中英文一一对应。

如果某个产品只需要到二级分类，没有三级分类，可以在 `G` 和 `H` 填 `/`。

不要把多个分类写在同一个单元格里。

不要在分类名前后加空格、换行或额外说明。

备注请放在 `K` 列，不要写进分类名称。

## 型号填写规则

有型号时填写在 `I` 列，例如：

```text
BD.3000
SL.3450
CH.5531
```

没有型号时可以留空或填 `无`。

同一个分类名下如果有型号，建议尽量填写。型号可以显著提高图片匹配准确率。

## 图片目录规范

图片按一级分类放入对应文件夹。

当前支持的图片目录包括：

```text
ziyuan/三级分类图片（2）/
ziyuan/三级类目图片/
ziyuan/8.拉手解决方案/
```

一级分类文件夹名称建议与 Excel 的一级中文分类一致，例如：

```text
滑轨解决方案
铰链解决方案
厨柜解决方案
衣柜解决方案
拉手解决方案
```

如果文件夹名称和 Excel 不一致，需要技术同事在 `resources/catalog-aliases.json` 中维护映射。

## 图片命名规范

推荐命名格式：

```text
三级中文名-型号.png
三级中文名-型号.jpg
```

示例：

```text
铝合金常规拉手-BD.3000.png
三节普通钢珠轨-SL.3450.png
缓冲铰链-CH.5531.png
```

如果没有型号，可以使用：

```text
三级中文名.png
```

示例：

```text
移门轨道.png
家具可调脚.png
```

## 图片文件要求

优先使用以下格式：

```text
.png
.jpg
.jpeg
.webp
```

建议使用白底、透明底或干净浅色底产品图。

图片中不要包含水印、报价、联系方式、二维码或无关文字。

同一个分类项只放一张主图。多张图后续可以扩展，但当前导入流程先按主图处理。

## 常见问题

表格名称和图片名称不一致会导致缺图。

例如：

```text
表格：不锈钢嵌入拉手
图片：不锈钢嵌入型拉手-BD.4413.png
```

这种情况可以通过别名解决，但更推荐从源头统一名称。

型号不一致也会影响匹配。

例如：

```text
表格：BD.3111
图片：带灯免拉-BD.3110.png
```

这种情况需要确认哪个型号是正确的。

拼写错误会影响英文路由和搜索。

例如：

```text
Lron handles&Knobs
```

这里可能是 `Iron handles&Knobs`。

## 交付前自查

交付 Excel 和图片前，请检查：

```text
一级分类是否属于 15 个固定分类
二级分类和三级分类是否完整
中文名和英文名是否一一对应
型号是否填在 I 列
图片是否放在正确一级分类文件夹
图片名是否使用“三级中文名-型号”
是否存在重复图片或过期图片
是否存在明显错别字
```

按这个规范维护，网站导入时的缺图和错配会明显减少。
